# GSL Academics Studio, Seed Pack

This is the kickoff pack for the GSL Academics Studio project. Drop this whole folder into `docs/seed/` in the new repo. Claude Code reads these on first session.

## Contents
1. `01-PRD.md` — Product brief. What we're building, why, for whom, what's in scope.
2. `02-CLAUDE.md` — Persistent context for Claude Code. Copy to repo root as `CLAUDE.md`.
3. `03-playbook-ppt-teaching-material.md` — Gold-standard playbook. Every other playbook copies this shape.
4. `04-skill-gsl-pedagogy-foundations.md` — Gold-standard skill file. Every other skill copies this shape.
5. `05-roadmap-phases.md` — Phase 1/2/3 deliverables, success criteria, what we're deliberately not building.
6. `06-gstack-execution-plan.md` — Step-by-step Claude Code commands, in order, empty repo to deployed Phase 1.

## Reading order
If you're Anish: 01 → 05 → 06 → 02 → 03 → 04.
If you're Claude Code on first session: read everything in numerical order.

## Conventions
- British English throughout. No emdashes anywhere.
- GSL brand colours and tone (see `02-CLAUDE.md`).
- All content is markdown. Seeds into Supabase on first deploy. Then edited in-app.

## Updates to the seed pack
Once the app is live, playbooks and skills are edited in-app through the rich editor. This seed pack is then frozen as the historical starting point. Exception: if you change the playbook or skill schema, update the gold-standard examples here and re-seed a fresh dev DB.
